BERT vs Generative AI

A business-focused comparison of transformer fine-tuning and zero-shot LLM classification for sentiment analysis.

Overview

Sentiment analysis is one of the most practical NLP tasks in business because it helps organisations turn unstructured customer feedback into usable signals. In this project, I compared two different approaches to binary sentiment classification: a fine-tuned BERT model and a zero-shot generative AI workflow using Llama 3. The goal was not just to compare model scores, but to evaluate which approach made more sense for a realistic business use case.

I framed the project around a streaming-platform scenario where customer reviews could be used for two purposes: identifying strongly negative feedback for service escalation and supporting content curation decisions. This made the work more than a technical model comparison. It became an exercise in choosing the right AI approach under business constraints.

Problem / Context

Streaming platforms receive large volumes of user reviews, and manually reading them is not scalable. An automated sentiment pipeline can help surface dissatisfied customers earlier and give decision-makers a more systematic view of audience response. In practice, that could support customer service teams, recommendation workflows, and catalogue management.

The key question behind this project was simple: should a business use a traditional fine-tuned NLP model or a large generative model for this kind of classification task? That decision affects cost, speed, data requirements, explainability, and deployment complexity.

Approach / Methodology

To explore this, I used the Rotten Tomatoes dataset for binary sentiment classification. I fine-tuned bert-base-uncased using the Hugging Face Trainer pipeline and compared it against Llama 3 running locally through Ollama in a zero-shot setting.

The BERT workflow required labelled data, tokenisation, training, and evaluation. The Llama 3 workflow relied on prompt design and post-processing rather than task-specific training. I evaluated both models using accuracy, precision, recall, and F1-score, while also considering practical trade-offs such as inference cost, scalability, flexibility, and privacy implications from local deployment.

What I did

  • Built a fine-tuning pipeline for BERT using Hugging Face and bert-base-uncased.
  • Designed a zero-shot prompting workflow for Llama 3 running locally through Ollama.
  • Evaluated both models using accuracy, precision, recall, and F1-score.
  • Framed the comparison around a realistic streaming-platform decision-support use case.
  • Reflected on ethical AI issues such as transparency, fairness, privacy, and deployment governance.

Results / Outcomes

Model Accuracy Precision Recall F1
BERT 0.830 0.811 0.835 0.823
Llama 3 0.930 0.934 0.914 0.924

Llama 3 outperformed BERT in this comparison, but the result should be interpreted carefully. BERT was evaluated on 500 examples, while Llama 3 was evaluated on 200 examples due to runtime constraints. The outcome suggests that zero-shot generative AI can perform strongly on practical classification tasks, especially when labelled data is limited, but it does not automatically make it the better production choice.

Why it matters

This project matters because many organisations are now deciding whether to use task-specific models or general-purpose LLMs for applied NLP workflows. The answer is rarely just “use the model with the best score.” A business also needs to consider labelled data availability, cost, latency, reproducibility, governance, and how the model will be used by real teams.

What I learned

This project changed the way I think about model selection. I went in expecting the fine-tuned classifier to win, but the generative model delivered stronger results in a low-data setting. That reinforced an important lesson: in applied analytics, the best model is the one that fits the business context, data availability, and operational constraints.

It also showed me that AI project evaluation should go beyond benchmark scores. Cost, reproducibility, interpretability, latency, and governance all matter when a model moves from notebook to business workflow.
